| Boundary | Description |
|----------|-------------|
| parsed config → FTP source monitor | Untrusted runtime flags decide whether FTP source runs once, polls, or should fail |
| monitor startup → user expectations | A successful `Start()` implies real ongoing sync behavior; false success is a correctness/security risk |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-02 | R | FTP source runtime semantics | mitigate | Add regression tests asserting the exact supported and rejected startup paths so future changes cannot reintroduce silent degradation |
| T-03-03 | D | long-running FTP source mode | accept | FTP uses polling rather than an event stream by design in v1; this is accepted because it preserves truthful one-way behavior without pretending event-native support |
</threat_model>
